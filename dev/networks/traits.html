Traits

Decisions.jl provides a variety of named traits on DecisionNetworks, ConditionalDists, and other objects in the package. They serve as hints to algorithms as well as the package itself.

This page provides the technical docs for each trait.

DecisionNetworks.DecisionsTrait — Type
abstract type DecisionsTrait

Abstract base trait type for all traits used in Decisions.jl.

source

Traits on decision networks

Sequentiality

DecisionNetworks.Sequentiality — Type
abstract type Sequentiality <: DecisionsTrait
Sequentiality(::DecisionNetwork)

Abstract trait type denoting whether agents act simultaneously or sequentially.

source
DecisionNetworks.Sequential — Type
Sequential <: Sequentiality

Trait indicating agents act sequentially (meaning each agent controls one or more independent decision nodes).

source
DecisionNetworks.Simultaneous — Type
Simultaneous <: Sequentiality

Trait indicating agents act simultaneously (meaning agents implement different indices of the same decision node(s)).

source

Partial observability / information perfection

DecisionNetworks.Observability — Type
abstract type Observability <: DecisionsTrait
Observability(::DecisionNetwork)

Abstract trait type indicating whether input to decision nodes passed to an agent or agents is "full," in that it is at least as informative as knowing the values of its predecessors.

source
DecisionNetworks.FullyObservable — Type
FullyObservable <: Observability

Trait indicating decision nodes receive perfect / full information.

source
DecisionNetworks.PartiallyObservable — Type
PartiallyObservable <: Observability

Trait indicating decision nodes do not receive perfect / full information.

Reward presence and style

DecisionNetworks.RewardStyle — Type
abstract type RewardStyle{ids} <: DecisionsTrait
RewardStyle(::DecisionNetwork)

Abstract trait indicating whether a decision network has a node analogous to a reward node: a node no others are conditioned on, typically for use in an objective. If so, indicates whether it represents a single or multiple reward, and the other nodes ids it is conditioned on.

source
DecisionNetworks.NoReward — Type
struct NoReward <: RewardStyle{()}

Trait indicating a decision network has no node analogous to a "reward" node.

source
DecisionNetworks.SingleReward — Type
SingleReward{ids} <: RewardStyle{ids}

Trait indicating a decision network has a "reward" node producing a single value, which is conditioned on the nodes ids.

source
DecisionNetworks.MultipleRewards — Type
MultipleRewards{ids} <: RewardStyle{ids}

Abstract trait indicating a decision network has a "reward" node producing multiple values, which is conditioned on the nodes ids.

The number of values produced can be known (DefiniteRewards) or unknown (IndefiniteRewards).

source
DecisionNetworks.DefiniteRewards — Type
DefiniteRewards{N, ids} <: MultipleRewards{ids}

Trait indicating a decision network has a "reward" node conditioned on ids which produces exactly N values (that is, it is a plate of size N).

source
DecisionNetworks.IndefiniteRewards — Type
IndefiniteRewards <: MultipleRewards{ids}

Trait indicating a decision network has a "reward" node conditioned on ids which is a plate of unknown size.

source

Traits on random variables

RVGroups can accept traits, which are primarily used as hints to sample and other such functions for optimal performance.

Due to their internal role they are not exported by default.

Terminality

DecisionNetworks.Terminality — Type
Terminality <: DecisionsTrait
Terminality(::RVGroup)

Abstract trait on a group of random variables indicating whether the corresponding distribution is expected to be able to produce Terminal() when sampled.

source
DecisionNetworks.Terminable — Type
Terminable <: Terminality

Trait indicating a group of random variables might produce Terminal() under certain values of its conditioning variables.

source
DecisionNetworks.NotTerminable — Type
NotTerminable <: Terminality

Trait indicating a group of random variables never produces Terminal() under any circumstances,.

source
DecisionNetworks.MaybeTerminable — Type
MaybeTerminal <: Terminality

Trait indicating a group of random variables may be able to produce Terminal() under some values of its conditioning variables, or for none of them.

source
